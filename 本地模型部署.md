# Ollama 本地模型集成到 LYAITEST 项目操作文档

## 概述
本文档记录如何在本机部署 Ollama 本地大模型，并集成到 LYAITEST 项目中，实现本地模型调用。

---

## 第1步：安装 Ollama

### Windows
1. 访问 https://ollama.com/download/windows
2. 下载安装包并运行
3. 安装完成后，打开终端验证：
```bash
ollama --version
预期输出：ollama version is x.x.x

macOS / Linux
bash
curl -fsSL https://ollama.com/install.sh | sh
ollama --version
第2步：下载本地模型
bash
ollama pull qwen2.5:1.5b
参数说明：

pull：下载模型

qwen2.5:1.5b：模型名称，约 986MB，适合本地运行

其他可选模型：

模型	大小	说明
qwen2.5:0.5b	约 350MB	最小，速度最快，能力最弱
qwen2.5:1.5b	约 986MB	平衡型，本教程使用
qwen2.5:7b	约 4.4GB	能力更强，需要更多内存
llama3.2:3b	约 2GB	Meta 模型，英文能力强
第3步：验证本地模型
命令行对话测试
bash
ollama run qwen2.5:1.5b "你好，请用一句话介绍你自己"
API 测试（Python）
python
import requests
import json

response = requests.post(
    "http://localhost:11434/api/generate",
    json={"model": "qwen2.5:1.5b", "prompt": "你好", "stream": False}
)
print(json.loads(response.text)["response"])
第4步：集成到 LYAITEST 项目
4.1 安装依赖
bash
pip install langchain-ollama
4.2 修改 app/services/agent.py
导入：

python
from langchain_ollama import ChatOllama
初始化本地模型（在 llm = ChatOpenAI(...) 附近添加）：

python
llm_local = ChatOllama(
    model="qwen2.5:1.5b",
    temperature=0.3,
    base_url="http://localhost:11434"
)
4.3 测试集成
bash
python -c "from app.services.agent import llm_local; print(llm_local.invoke('用一句话介绍你自己').content)"
预期输出：本地模型的回答

第5步：切换模型
场景	推荐模型
开发测试，速度快	qwen2.5:1.5b
需要更高质量回答	qwen2.5:7b
英文任务	llama3.2:3b
云端大模型（生产）	deepseek-chat / glm-4
切换方式：

python
llm_local = ChatOllama(
    model="qwen2.5:7b",  # 改成你需要的模型
    temperature=0.3
)
常见问题
问题	解决方案
ollama: command not found	Ollama 未安装或未添加到 PATH，重新安装
port 11434 already in use	已有 Ollama 进程在运行，直接使用即可
ModuleNotFoundError: No module named 'langchain_ollama'	执行 pip install langchain-ollama
模型回答质量低	换更大的模型（如 7b）或调整 temperature
本地模型速度慢	使用更小的模型（如 0.5b）或增加系统内存
